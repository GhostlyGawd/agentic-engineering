# Phase 1.5: MCP Connection & Integration Defects Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the `agentic-graph` MCP server actually connect to Claude Code in real use (it never has), close the integration gaps that hid this, and re-establish the live review-loop e2e on a deterministic footing.

**Architecture:** The fix is "self-resolving registration": `init_project` writes a `.mcp.json` whose `command` is the absolute path of the interpreter that ran it (`sys.executable` = the venv python), so the server is always launchable without assuming anything is on PATH. The repo's own `.mcp.json` becomes a generated, git-ignored artifact. The e2e harness is hardened to (a) stage that same registration into its temp project, (b) decode CLI output as UTF-8, and (c) not leak orphaned child processes. Every defect is dogfic-logged as a `Retro`.

**Tech Stack:** Python 3.12, `mcp==1.27.1`, pytest (markers: `llm` for live-agent gate), PowerShell 5.1 hook, Claude Code plugin (`.mcp.json`, `commands/*.md`).

## Background: the confirmed diagnosis (read first)

Investigation on 2026-05-19 established, with evidence:

1. **The `agentic-graph` MCP server had never connected to Claude Code.** Repo `.mcp.json` used a bare `"command": "agentic-mcp"`; that console script lives only in `mcp-server/.venv/Scripts/` and is not on PATH, so `claude mcp list` reported `agentic-graph - Failed to connect` in every session.
2. **Tests bypassed the real launch path.** `mcp-server/tests/test_server.py` spawns the server via `sys.executable -m agentic_mcp.server` (resolved, works); other tests call handlers in-process. The shipped `.mcp.json` command was never exercised — contradicting design §11's "production and test exercise the same code path".
3. **The live-loop "flakiness" was downstream of #1.** With the server connected (command fixed + staged into the working dir), a headless review-pr probe flipped from "improvise, log 0 findings" to driving the real tools: `mcp__agentic-graph__get_node` -> pytest gate -> `query_graph` -> `log_finding`. Verified twice.
4. **`/agentic:init` does not register the server.** `init_project.run()` scaffolds `.agentic/` but writes no `.mcp.json`; `commands/init.md` calls `agentic-mcp-init` (also venv-local). A tool-bootstrapped project therefore has no tools.
5. **Plugin commands aren't loaded in a bare headless temp dir.** The probe agent reported "the `/agentic:review-pr` slash command isn't installed here" and read the agent `.md` files manually.
6. **`claude -p` hangs on agent-spawned background tasks.** A stray background search kept the process alive ~13.5 min after the real turn ended — the true cause of the original 900s timeout (not a slow review).
7. **Harness output-decode bug.** `run_claude_headless` uses `subprocess.run(..., text=True)` with no `encoding`, so non-cp1252 bytes (e.g. `->` U+2192) in CLI output crash the read on this machine.

Recurring bug-class: bare command names that aren't PATH-resolvable (also fixed earlier in `llm_harness.py` for the `claude` shim itself, commit `44804ba`).

```python
# in mcp-server/src/agentic_mcp/init_project.py
import sys  # add to imports

def _write_mcp_config(root: Path) -> None:
    """Register agentic-graph using the CURRENT interpreter's absolute path.

    sys.executable is the venv python that ran agentic-mcp-init, so the command
    is always resolvable on this machine without relying on PATH. Merge into any
    existing .mcp.json rather than clobbering other servers.
    """
    mcp_path = root / ".mcp.json"
    if mcp_path.exists():
        try:
            cfg = json.loads(mcp_path.read_text(encoding="utf-8"))
        except (ValueError, OSError):
            cfg = {}
    else:
        cfg = {}
    if not isinstance(cfg, dict):
        cfg = {}
    servers = cfg.setdefault("mcpServers", {})
    servers["agentic-graph"] = {
        "command": sys.executable,
        "args": ["-m", "agentic_mcp.server"],
        "env": {"AGENTIC_DB_PATH": str((root / ".agentic" / "graph.db").resolve())},
    }
    mcp_path.write_text(json.dumps(cfg, indent=2) + "\n", encoding="utf-8")
```

Then call it at the end of `run()` (after `compat_path` is written):

```python

```

```python
import json, sys  # ensure imported at top

def stage_mcp_config(project, db_path) -> "Path":
    from pathlib import Path
    cfg = {"mcpServers": {"agentic-graph": {
        "command": sys.executable,
        "args": ["-m", "agentic_mcp.server"],
        "env": {"AGENTIC_DB_PATH": str(db_path)},
    }}}
    p = Path(project) / ".mcp.json"
    p.write_text(json.dumps(cfg, indent=2), encoding="utf-8")
    return p


def run_claude_headless(prompt: str, cwd, timeout: int = 900, mcp_config=None) -> dict:
    exe = _claude_exe()
    cmd = [exe, "-p", prompt, "--output-format", "json",
           "--permission-mode", "bypassPermissions"]
    if mcp_config is not None:
        cmd += ["--mcp-config", str(mcp_config), "--strict-mcp-config"]
    proc = subprocess.run(
        cmd, cwd=str(cwd), capture_output=True, text=True,
        encoding="utf-8", errors="replace", timeout=timeout,
    )
    if proc.returncode != 0:
        raise RuntimeError(
            f"claude -p exited {proc.returncode}\n"
            f"stdout:\n{proc.stdout[-2000:]}\nstderr:\n{proc.stderr[-2000:]}"
        )
    return json.loads(proc.stdout)
```

```python
import subprocess as _sp

def _kill_tree(pid: int) -> None:
    try:
        _sp.run(["taskkill", "/F", "/T", "/PID", str(pid)],
                capture_output=True, text=True)
    except Exception:
        pass
```

Wrap the `subprocess.run` call (switch to Popen so we hold the pid):

```python
    proc = subprocess.Popen(
        cmd, cwd=str(cwd), stdout=subprocess.PIPE, stderr=subprocess.PIPE,
        text=True, encoding="utf-8", errors="replace",
    )
    try:
        out, err = proc.communicate(timeout=timeout)
    except subprocess.TimeoutExpired:
        _kill_tree(proc.pid)
        proc.communicate()
        raise
    if proc.returncode != 0:
        raise RuntimeError(f"claude -p exited {proc.returncode}\nstdout:\n{out[-2000:]}\nstderr:\n{err[-2000:]}")
    return json.loads(out)
```
